**Date**: 2026-01-21  
**Status**: Open  
**Priority**: Low  
**Source**: [OCP Protocol Review](./2026.01.21-ocp-protocol-review.md) â€” Issue 6.1

## Problem

Many public methods lack JSDoc documentation, making the SDK harder to use without reading source
code.

## Recommendation

Add JSDoc to all public API methods with:

- Description
- `@param` for each parameter
- `@returns` description
- `@throws` for known error types
- `@example` for common use cases

## Tasks

- [ ] Identify all public API methods in `OcpClient`
- [ ] Add JSDoc to `OcpClient` constructor and methods
- [ ] Add JSDoc to batch builder methods
- [ ] Add JSDoc to utility functions
- [ ] Configure TypeDoc to generate API docs
- [ ] Add @example blocks for common operations

## Acceptance Criteria

- All public methods have JSDoc
- TypeDoc generates complete API documentation
- Examples cover common use cases
- No "any" types in public signatures
